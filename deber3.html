¿Qué es la Programación Orientada a Objetos (POO)?

La Programación Orientada a Objetos (POO) es un paradigma de programación que organiza el código en objetos que contienen tanto datos (atributos) como comportamientos (métodos).

Este enfoque permite:

  • Modelar el mundo real: Representar entidades reales como objetos en el código
  • Reutilización de código: Crear componentes que pueden usarse en múltiples lugares
  • Mantenibilidad: Código más organizado y fácil de modificar
  • Escalabilidad: Facilita el crecimiento de aplicaciones complejas

Concepto clave: La POO se basa en modelar el software como una colección de objetos que interactúan entre sí, en lugar de una simple lista de instrucciones.

Los 4 Pilares de la POO

La POO se fundamenta en cuatro principios esenciales que definen su estructura y funcionamiento:

Encapsulamiento

Ocultar los detalles internos de un objeto y exponer solo lo necesario

Herencia

Crear nuevas clases basadas en clases existentes, heredando sus características

Polimorfismo

Permitir que objetos de diferentes clases respondan al mismo mensaje de formas distintas

Abstracción

Representar características esenciales sin incluir detalles complejos de implementación

1. Encapsulamiento

El encapsulamiento es el mecanismo que agrupa datos y métodos relacionados en una unidad (objeto) y controla el acceso a ellos. Protege los datos internos de modificaciones no autorizadas.

Características:

  • Oculta la implementación interna del objeto
  • Expone solo una interfaz pública para interactuar
  • Protege los datos mediante modificadores de acceso (público, privado, protegido)
  • Previene el acceso directo a los atributos desde fuera de la clase

Ejemplo:

class CuentaBancaria { // Atributo privado (encapsulado) #saldo = 0; constructor(saldoInicial) { this.#saldo = saldoInicial; } // Método público para acceder al saldo getSaldo() { return this.#saldo; } // Método público para depositar depositar(cantidad) { if (cantidad > 0) { this.#saldo += cantidad; return true; } return false; } // Método público para retirar retirar(cantidad) { if (cantidad > 0 && cantidad <= this.#saldo) { this.#saldo -= cantidad; return true; } return false; } } // Uso const cuenta = new CuentaBancaria(1000); cuenta.depositar(500); console.log(cuenta.getSaldo()); // 1500

2. Herencia

La herencia permite crear nuevas clases (clases hijas) basadas en clases existentes (clases padre), heredando sus atributos y métodos. Promueve la reutilización del código.

Características:

  • La clase hija hereda atributos y métodos de la clase padre
  • Permite especializar comportamientos en las clases hijas
  • Establece relaciones "es un" (ejemplo: un Perro es un Animal)
  • Facilita la extensión del código sin modificar las clases existentes

Ejemplo:

// Clase padre class Animal { constructor(nombre) { this.nombre = nombre; } hacerSonido() { console.log(`${this.nombre} hace un sonido`); } } // Clase hija que hereda de Animal class Perro extends Animal { constructor(nombre, raza) { super(nombre); // Llama al constructor del padre this.raza = raza; } // Sobrescribe el método del padre hacerSonido() { console.log(`${this.nombre} dice: ¡Guau!`); } // Método específico de Perro moverCola() { console.log(`${this.nombre} mueve la cola feliz`); } } // Uso const miPerro = new Perro("Max", "Labrador"); miPerro.hacerSonido(); // Max dice: ¡Guau! miPerro.moverCola(); // Max mueve la cola feliz

3. Polimorfismo

El polimorfismo permite que objetos de diferentes clases respondan de manera diferente al mismo mensaje o método. "Poli" significa muchas y "morfismo" significa formas.

Características:

  • Una misma interfaz puede comportarse de diferentes formas
  • Permite tratar objetos de diferentes clases de manera uniforme
  • Facilita la extensibilidad del código
  • Se logra principalmente mediante la sobrescritura de métodos

Ejemplo:

class Figura { calcularArea() { return 0; } } class Rectangulo extends Figura { constructor(base, altura) { super(); this.base = base; this.altura = altura; } // Implementación específica para Rectángulo calcularArea() { return this.base * this.altura; } } class Circulo extends Figura { constructor(radio) { super(); this.radio = radio; } // Implementación específica para Círculo calcularArea() { return Math.PI * this.radio ** 2; } } // Polimorfismo en acción const figuras = [ new Rectangulo(5, 3), new Circulo(4) ]; figuras.forEach(figura => { console.log(`Área: ${figura.calcularArea()}`); }); // Cada figura calcula su área de forma diferente

4. Abstracción

La abstracción consiste en representar las características esenciales de un objeto sin incluir los detalles complejos de implementación. Se enfoca en "qué hace" en lugar de "cómo lo hace".

Características:

  • Oculta la complejidad interna y muestra solo lo necesario
  • Define interfaces y contratos que las clases deben cumplir
  • Permite trabajar con conceptos de alto nivel
  • Facilita el diseño y comprensión del sistema

Ejemplo:

// Clase abstracta (concepto) class Vehiculo { constructor(marca, modelo) { this.marca = marca; this.modelo = modelo; } // Método abstracto (debe ser implementado por clases hijas) arrancar() { throw new Error("Método 'arrancar' debe ser implementado"); } // Método concreto mostrarInfo() { console.log(`${this.marca} ${this.modelo}`); } } class Auto extends Vehiculo { // Implementa el método abstracto arrancar() { console.log("Girando la llave... El auto arranca"); } } class Moto extends Vehiculo { // Implementa el método abstracto de forma diferente arrancar() { console.log("Presionando el botón... La moto arranca"); } } // Uso const miAuto = new Auto("Toyota", "Corolla"); miAuto.arrancar(); // Girando la llave... const miMoto = new Moto("Honda", "CBR"); miMoto.arrancar(); // Presionando el botón...

Clases y Objetos

¿Qué es una Clase?

Una clase es una plantilla o molde que define las características (atributos) y comportamientos (métodos) que tendrán los objetos creados a partir de ella.

¿Qué es un Objeto?

Un objeto es una instancia de una clase. Es una entidad concreta creada a partir del molde que representa la clase.

Clase

  • Es un modelo o plantilla
  • Define estructura y comportamiento
  • Es abstracta (no ocupa memoria)
  • Ejemplo: Receta de pastel

Objeto

  • Es una instancia concreta
  • Tiene valores específicos
  • Ocupa memoria
  • Ejemplo: El pastel ya hecho

Ejemplo Completo:

// CLASE: Plantilla para crear estudiantes class Estudiante { // Constructor: inicializa los atributos constructor(nombre, edad, carrera) { this.nombre = nombre; // Atributo this.edad = edad; // Atributo this.carrera = carrera; // Atributo this.notas = []; // Atributo } // Método: comportamiento del estudiante agregarNota(nota) { this.notas.push(nota); } // Método: calcular promedio calcularPromedio() { if (this.notas.length === 0) return 0; const suma = this.notas.reduce((a, b) => a + b, 0); return suma / this.notas.length; } // Método: mostrar información mostrarInfo() { console.log(`Estudiante: ${this.nombre}`); console.log(`Edad: ${this.edad}`); console.log(`Carrera: ${this.carrera}`); console.log(`Promedio: ${this.calcularPromedio()}`); } } // OBJETOS: Instancias concretas de la clase Estudiante const estudiante1 = new Estudiante("María García", 20, "Ingeniería"); estudiante1.agregarNota(85); estudiante1.agregarNota(90); estudiante1.agregarNota(88); const estudiante2 = new Estudiante("Juan Pérez", 22, "Medicina");
